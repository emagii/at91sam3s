00030 /**
00031  * \file
00032  *
00033  * \section Purpose
00034  *
00035  * The INTERL %norflash Low-level driver code implement procedures to program
00036  * basic operations described INTEL-specified command set flash devices.
00037  * The various commands recognized by the devices are listed in the Commands
00038  * Tables provided in the corresponding INTEL command set compatible flash
00039  * datasheets. All operation functions are blocked, they wait for the
00040  * completion of an operation by polling the status register.
00041  *
00042  * \section Usage
00043  * -# Flash program using INTEL_Write_Data().
00044  *    - The Program command is used to modify the data stored at the
00045  *      specified device address. Programming can only change bits
00046  *      from ¡®1¡¯ to ¡®0¡¯. It may be necessary to erase the block before
00047  *      programming to addresses within it. Programming modifies a single
00048  *      Word at a time using static function intel_Program(). Programming
00049  *      larger amounts of data must be done in one Word at a time by
00050  *      giving a Program command, waiting for the command to complete,
00051  *      giving the next Program command and so on.
00052  * -# erase a block within the flash using INTEL_EraseSector().
00053  *    - Flash erase is performed on a block basis. An entire block is
00054  *      erased each time an erase command sequence is given.
00055  * -# erase whole blocks within the flash using INTEL_EraseChip().
00056  * -# INTEL_Reset() function can be issued, between Bus Write cycles
00057  *    before the start of a program or erase operation, to return the
00058  *    device to read mode.
00059  * -# INTEL_ReadDeviceID() is used to retrieve information
00060  *    about the Flash Device type.
00061  * -# INTEL_ReadManufactoryId() is used to retrieve information
00062  *    about the Flash Device Manufactory ID.
00063  */
