00037 /** \addtogroup wdt_module Working with WDT
00038  * The WDT driver provides the interface to configure and use the WDT
00039  * peripheral.
00040  *
00041  * The WDT can be used to prevent system lock-up if the software becomes
00042  * trapped in a deadlock. It can generate a general reset or a processor
00043  * reset only. It is clocked by slow clock divided by 128.
00044  *
00045  * The WDT is running at reset with 16 seconds watchdog period (slow clock at 32.768 kHz)
00046  * and external reset generation enabled. The user must either disable it or
00047  * reprogram it to meet the application requires.
00048  *
00049  * To use the WDT, the user could follow these few steps:
00050  * <ul>
00051  * <li>Enable watchdog with given mode using \ref WDT_Enable().
00052  * <li>Restart the watchdog using \ref WDT_Restart() within the watchdog period.
00053  * </ul>
00054  *
00055  * For more accurate information, please look at the WDT section of the
00056  * Datasheet.
00057  *
00058  * \note
00059  * The Watchdog Mode Register (WDT_MR) can be written only once.\n
00060  *
00061  * Related files :\n
00062  * \ref wdt.c\n
00063  * \ref wdt.h.\n
00064  */
