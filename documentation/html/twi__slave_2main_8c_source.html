00030 /**
00031  * \page twi_slave TWI Slave Example
00032  *
00033  * \section Purpose
00034  *
00035  * This project demonstrates the TWI peripheral in slave mode. It mimics the
00036  * behavior of a serial memory, enabling the TWI master to read and write
00037  * data in its internal SRAM.
00038  *
00039  * \section Requirements
00040  *
00041  * In addition, another device will be needed to act as the TWI master. The
00042  * twi_eeprom example can be used for that, in which case a second kit
00043  * supported by that project is needed.
00044  * -# Connect TWD1 (SDA) for the 2 boards: PB4(pin 51) of chipset socket pin.
00045  * -# Connect TWCK1 (SCL) for the 2 boards: PB5(pin 76) of chipset socket pin.
00046  * -# Connect GND for the 2 boards: GND on extension header.
00047  * -# Add a pull up of 2,2KOhms on TWD and TWCK
00048  *
00049  * By default, PB4 and PB5 are SWJ-DP pins, it can be used as a pin for TWI1
00050  * peripheral in the end application. Mode selection between SWJ-DP (System IO mode)
00051  * and general IO mode is performed through the AHB Matrix CCFG_SYSIO.
00052  *
00053  * \section Description
00054  *
00055  * After launching the program, the device will act as a simple TWI-enabled
00056  * serial memory containing 512 bytes. This enables this project to be used
00057  * with the basic-twi-eeprom-project as the master.
00058  *
00059  * To write in the memory, the TWI master must address the device first, then
00060  * send two bytes containing the memory address to access. Additional bytes are
00061  * treated as the data to write.
00062  *
00063  * Reading is done in the same fashion, except that after receiving the memory
00064  * address, the device will start outputting data until a STOP condition is
00065  * sent by the master.
00066  * The default address for the TWI slave is fixed to Ox50. If the board has a TWI
00067  * component with this adress, you can change the define AT24C_ADDRESS in main.c
00068  * of twi_eeprom example, and the define SLAVE_ADDRESS in main.c of twi_slave
00069  * exmaple.
00070  *
00071  * \section Usage
00072  *
00073  * -# Build the program and download it inside the evaluation board. Please
00074  *    refer to the
00075  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6224.pdf">
00076  *    SAM-BA User Guide</a>, the
00077  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00078  *    GNU-Based Software Development</a> application note or to the
00079  *    <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00080  *    IAR EWARM User Guide</a>, depending on your chosen solution.
00081  * -# On the computer, open and configure a terminal application
00082  *    (e.g. HyperTerminal on Microsoft Windows) with these settings:
00083  *   - 115200 bauds
00084  *   - 8 bits of data
00085  *   - No parity
00086  *   - 1 stop bit
00087  *   - No flow control
00088  * -# Start the application.
00089  * -# Upon startup, the application will output the following line on the DBGU:
00090  *    \code
00091  *     -- twi slave example xxx --
00092  *     -- SAM3S-EK
00093  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00094  *     -I- Configuring the TWI in slave mode
00095  *    \endcode
00096  * -# For the TWI Master board, see the description inside his project
00097  * -# and the "Master" board will output:
00098  *    \code
00099  *     -- twi eeprom example xxx --
00100  *     -- SAM3S-EK
00101  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00102  *     -I- Filling page #0 with zeroes ...
00103  *     -I- Filling page #1 with zeroes ...
00104  *     -I- Read/write on page #0 (polling mode)
00105  *     -I- 0 comparison error(s) found
00106  *     -I- Read/write on page #1 (IRQ mode)
00107  *     -I- Callback fired !
00108  *     -I- Callback fired !
00109  *     -I- 0 comparison error(s) found
00110  *    \endcode
00111  *
00112  * \section References
