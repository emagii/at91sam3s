00030 /**
00031  * \page eefc_pgm EEFC programming example
00032  *
00033  * \section Purpose
00034  * This basic example shows how to use the Enhance Embedded Flach (EEFC) peripheral
00035  * available on the newest Atmel SAM3S microcontrollers. It details steps required to
00036  * program the internal flash, and manage secure and lock bits.
00037  *
00038  * \section Requirements
00039  *
00040  * This package can be used with SAM3S evaluation kits.
00041  *
00042  * \section Description
00043  * The SAM3S ROM code embeds small In Application Programming Procedure.
00044  * Since this function is executed from ROM, this allows Flash programming (such as sector write)
00045  * to be done by code running in Flash.\n
00046  * The IAP function entry point is retrieved by reading the NMI vector in ROM ().
00047  * This function takes two argument in parameter: bank index (always 0) and the command to be sent to the EEFC.
00048  *    \code
00049  *    static uint32_t  (*IAP_PerformCommand)(uint32_t, uint32_t);
00050  *    IAP_PerformCommand = (uint32_t (*)(uint32_t, uint32_t)) *((uint32_t *) 0x00800008);
00051  *    IAP_PerformCommand(0, (0x5A << 24) | (argument << 8) | command);
00052  *     \endcode
00053  * IAP function returns the value of the MC_FSR register.
00054  * The required steps are:
00055  * - Unlock a page.
00056  * - Program a page of the embedded flash with incremental values (0x0, 0x1, 0x2, 0x3бн.) by using the IAP function.
00057  * - Check the flash is correctly programmed by reading all the values programmed.
00058  * - Lock the page.
00059  * - Set the security bit.
00060  *
00061  * The SAM3S features a security bit, based on a specific General Purpose NVM bit 0.
00062  * When the security is enabled, any access to the Flash, SRAM, Core Registers and Internal
00063  * Peripherals either through the ICE interface is forbidden.
00064  * This example will reproduce this scene.
00065  *
00066  * \section Usage
00067  *
00068  * -# Build the program and download it inside the evaluation board. Please
00069  *    refer to the
00070  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6224.pdf">
00071  *    SAM-BA User Guide</a>, the
00072  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00073  *    GNU-Based Software Development</a> application note or to the
00074  *    <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00075  *    IAR EWARM User Guide</a>, depending on your chosen solution.
00076  * -# On the computer, open and configure a terminal application
00077  *    (e.g. HyperTerminal on Microsoft Windows) with these settings:
00078  *   - 115200 bauds
00079  *   - 8 bits of data
00080  *   - No parity
00081  *   - 1 stop bit
00082  *   - No flow control
00083  * -# Start the application.
00084  * -# In the terminal window, the following text should appear:
00085  *    \code
00086  *     -- EEFC Programming Example xxx --
00087  *     -- xxxxxx-xx
00088  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00089  *     -I- Unlocking last page
00090  *     -I- Writing last page with walking bit pattern
00091  *     -I- Checking page contents .............................................................. ok
00092  *     -I- Locking last page
00093  *     -I- Try to program the locked page...
00094  *     -I- Please open Segger's JMem program
00095  *     -I- Read memory at address 0x0043FF00 to check contents
00096  *     -I- Press any key to continue...
00097  *     -I- Good job!
00098  *     -I- Now set the security bit
00099  *     -I- Press any key to continue to see what happened...
00100  *     -I- Setting GPNVM #0
00101  *     -I- All tests done
00102  * \endcode
